Datasets Overview

We provide the complete datasets obtained from the Sourcerer. The datasets as very large file sizes, are publicly available, and can be downloaded/accessed by the public. They can be easily downloaded with BTSync, for which we provide a public hash key.
The repositories represent the same information in different formats, after different phases of preprocessing. Under Tutorials you can learn how to replicate this phases. In total we represent 18,000 Java projects downloaded from Apache, Java.net, Google Code and Sourceforge as of Summer 2001. Here is a brief description of them:

The filtered files column contains the number of files that are actually used by Sourcerer's feature extractor when performing the analysis. The discrepancy between this column and the files column is primarily caused by files duplicated due to our automated download process.

The raw dataset

This dataset, called sourcerer_repo_2011, takes 368Gb compressed or 433Gb uncompressed, and is a collection of 74,351 Java projects and assorted jar files collected from Google Code, SourceForge and Apache as of Summer 2011. It is also integrated in an infrastructure being built by the DARPA MUSE program, which will hopefully be available for other researchers sometime in the future.
  • BTSync hash: BLSPQKT7JO6S37ZFKA5VX6LF6ACNPFQJF

The extracted dataset

This extracted repository, called sourcerer_repo_2011_extracted, takes 84Gb compressed or 92Gb uncompressed, and is an intermediary artifact produced from the raw dataset sourcerer_repo_2011, which contains facts (fully qualified names, metrics, etc.) about each file of each project. We also provide a server serving this data. Please refer to Database to know more.
  • BTSync hash: BU6XBJ3BSB56AAMU5UVMH5JD5HCZ7VQPV

The relational database

This database, called sourcerer_repo_2011_database, takes 24Gb as a compressed MySQL dump or 448Gb when restored, and is an artifact built from the extracted facts (sourcerer_repo_2011_extracted). This database contains the output of parsing and decomposing the source code into code entities and their relations, and it is provided to facilitate static analysis.
  • BTSync hash: BRX4URA35D2CHSV3EJXBZB65Z6PX23HOP
